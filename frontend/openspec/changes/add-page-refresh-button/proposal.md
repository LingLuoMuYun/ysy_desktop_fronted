# 增加页面状态刷新按钮

## 背景

用户希望优化前端中 AI 助手选择与 AI 对话相关入口，先在全局顶部栏增加刷新入口：首页放在“打开位置”按钮左侧，其他页面放在“AI 助手”按钮左侧，点击后刷新当前页面状态。

## 目标

- 在首页顶部栏“打开位置”左侧增加刷新按钮。
- 在项目、任务、数据、模型、设置页顶部栏“AI 助手”左侧增加刷新按钮。
- 点击刷新按钮后重新挂载当前页面内容，使页面内加载逻辑和局部 UI 状态重新初始化。

## 非目标

- 不修改后端接口。
- 不新增全局数据缓存层。
- 不清空首页由 `AppShell` 托管的历史对话列表。
- 不改变 AI 助手打开/关闭状态和右侧栏宽度状态。

## 影响范围

- `frontend/src/layouts/AppShell.tsx`
- `frontend/src/layouts/WindowTitleBar.tsx`
- `frontend/src/styles/globals.css`

## 风险等级

低风险。该变更只增加前端只读刷新入口，不执行本机高风险动作，不修改后端状态。

## 验收标准

- 首页顶部栏“打开位置”左侧出现刷新按钮。
- 非首页顶部栏“AI 助手”左侧出现刷新按钮。
- 点击刷新按钮后当前页面内容重新挂载，设置页等带加载逻辑的页面会重新触发数据加载。
- `pnpm run typecheck` 通过。
