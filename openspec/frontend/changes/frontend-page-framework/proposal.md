# Proposal: frontend-page-framework

## 背景

前端即将进入页面框架搭建阶段。当前 `frontend/` 仅有说明文件，尚未初始化 React + Electron 页面结构。用户已提供原型地址 `http://10.0.89.199:5173`，本次变更需要优先保证页面样式和结构与原型一致，再进入后续业务联调。

前端 OpenSpec 相关内容统一放在 `/Users/lingluo/Desktop/ysy_desktop-main/openspec/frontend/` 下，本变更位于 `openspec/frontend/changes/frontend-page-framework/`。

## 目标

- 初始化前端页面框架所需的 React + TypeScript + Electron + Vite 基础结构。
- 搭建桌面应用主外壳：桌面窗口容器、左侧导航、统一顶部栏、主工作区和右侧上下文栏。
- 按原型还原六个一级页面：首页、项目、任务、数据、模型、设置。
- 首页还原居中 AI 对话入口、模型选择、项目关联、建议问题和右侧资源概览入口。
- 项目、任务、数据、模型、设置页面还原高密度工作台布局，包括顶部模块名、筛选、列表/卡片、状态标签、操作入口和右侧 AI 助手面板。
- 保留 loading、empty、error、disabled、confirm placeholder 等基础状态展示位置，便于后续接入真实数据和操作。

## 非目标

- 不接入真实后端 API。
- 不实现真实训练、推理服务、环境检测、数据检查、模型检查或文件操作。
- 不实现真实 AI 对话、AI 诊断或 `action_request` 执行。
- 不新增一级模块，不改变首页、项目、任务、数据、模型、设置六大模块边界。
- 不实现 Git 仓库、branch、commit、diff 等 V1.0 非目标能力。
- 不确定最终 Electron 打包、自动更新和发布策略。

## 影响范围

- 前端代码：`frontend/` 下的工程初始化、路由、布局、页面、组件、样式、mock 数据和基础测试。
- OpenSpec：仅新增本 Change，不修改 Global Spec 或 Module Spec。
- 文档：必要时更新 `frontend/README.md` 的启动和验证方式。

## 用户流程

1. 用户打开桌面智算前端。
2. 应用展示原型一致的桌面窗口式主界面。
3. 用户通过左侧导航切换首页、项目、任务、数据、模型和设置。
4. 首页显示中心 AI 输入入口，可从顶部栏打开右侧本机资源概览。
5. 其他模块显示对应业务列表、筛选区、操作入口和右侧 AI 助手，一级模块名显示在统一顶部栏。
6. 用户可以看到各页面的静态结构、状态和占位反馈，但不会触发真实本机操作。

## 风险

- 原型页面可能继续变化，实施时需要以用户确认的当前原型为准。
- 当前仓库已有未提交 OpenSpec 迁移痕迹，实施和提交时必须避免误回滚用户已有改动。
- 页面框架阶段容易把 mock 操作误认为真实功能，必须在 UI 和代码边界中保持占位状态。
- 高风险操作入口必须保留确认弹窗或禁用占位，不能直接执行任何本机动作。

## 验收标准

- 前端 OpenSpec Change 文件均位于 `openspec/frontend/changes/frontend-page-framework/`。
- 页面框架可启动并访问。
- 六个一级页面均可从左侧导航进入。
- 首页布局与原型的中心对话入口和右侧资源概览结构一致。
- 项目、任务、数据、模型、设置页面均有与原型一致的顶部模块名、筛选、列表/卡片、状态标签和操作入口。
- 非首页页面展示右侧 AI 助手面板或等价占位。
- 所有真实本机操作保持 mock、disabled 或确认占位，不产生实际副作用。
- 通过静态检查、构建检查和至少一次浏览器视觉验证。
