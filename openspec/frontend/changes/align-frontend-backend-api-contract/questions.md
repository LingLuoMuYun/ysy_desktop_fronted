# Questions

## 已确认

1. 后端新版文档作为前端项目模块业务 API 的唯一准入标准；旧前端接口替换文档与其冲突时，全部以后端新版为准。
2. 项目编辑功能保留，但当前阶段暂不处理；前端只预留位置，不实现旧版 `PATCH /api/projects/{projectId}` 请求。
3. 当前阶段只针对项目模块，不处理安装初始化、模型下载、任务、数据、模型、环境、个人设置等其他模块问题。
4. 项目列表做分页处理：每页展示 5 个项目；底部展示当前页、总页数和总项目数。`totalPages` 若后端返回则使用，若未返回则由 `total` 和 `pageSize=5` 计算。
5. 项目创建按后端项目章节处理：必须传 `confirmed=true`，当前不额外要求 `Idempotency-Key`。
6. 项目工作区路径提交前必须调用路径校验接口。
7. 项目编辑入口先预留一个空按钮，不触发请求。
8. 项目详情页中的关联数据、任务、模型 Tab 本阶段保留空态。
