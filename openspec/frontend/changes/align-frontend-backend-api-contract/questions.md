# Questions

## P0 必须确认后才能实现

1. 后端新版文档是否作为前端业务 API 的唯一准入标准？旧前端接口替换文档与其冲突时，是否全部以后端新版为准？
2. 项目编辑能力是否取消？如果保留，请后端补充 `PATCH /api/projects/{projectId}` 的请求体、响应体和风险等级。
3. 安装初始化接口 6.1-6.12 在前端归属哪个入口？是设置页新增“初始化/能力安装”区域，还是作为首次启动引导流程？
4. 模型下载接口 5.8-5.13 是否进入 V1.0 当前前端范围？如果进入，入口在模型页还是任务创建流程中的能力补齐步骤？
5. 所有创建类接口是否都要求 `confirmed=true`？后端文档中明确项目创建、任务启动、环境创建等需要确认，但数据/模型导入等中风险记录类动作是否也统一要求确认？
6. `Idempotency-Key` 统一放请求头，还是允许请求体 `idempotencyKey`？前端是否优先使用请求头并在少数接口兜底请求体？
7. 业务 API base URL 是否统一使用 `VITE_API_BASE_URL`，还是保留环境业务后端 `VITE_ENVIRONMENTS_API_BASE_URL` 的独立配置？
8. 桌面端本地目录选择优先用 Electron IPC 还是后端 `POST /api/local-files/pick-directory`？两者同时存在时哪个是正式入口？

## P1 实现前确认

1. 分页响应是否所有列表都统一包含 `page`、`pageSize`、`total`、`totalPages`、`hasMore`？项目列表示例缺少 `totalPages`，前端是否按可选处理？
2. 状态展示是否所有资源都采用 `status` / `statusText` 分离？任务状态文档使用 `apiStatus`，列表输出字段是否仍叫 `status`？
3. `CheckResult` 是否作为数据、模型、环境、任务预检和部署兼容性唯一检查结果结构？
4. 删除类接口默认只删登记记录时，是否都需要 `deleteLocalFiles=false`？数据、模型、任务的本地文件删除能力是否 V1.0 禁用？
5. 错误码是否全部小写 snake_case？任务执行错误如 `COMMAND_NOT_FOUND` 是否只出现在业务字段 `errorCode`，不作为统一 `error.code`？
6. 任务日志分页参数和排序字段是否固定？前端日志视图需要支持尾部实时刷新还是只支持分页查询？
7. 任务报告生成是否必须使用幂等键？生成失败是否只改变 Report 状态，不改变 Task 主状态？
8. 部署任务健康检查通过后持续 `running`，前端是否需要单独展示服务健康状态字段？

5. 模型候选下载前的硬件评估结果是否可复用安装初始化的硬件检测结果？
