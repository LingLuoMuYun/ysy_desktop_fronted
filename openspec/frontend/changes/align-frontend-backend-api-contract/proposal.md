# Align Frontend To Backend API Contract

## 背景

后端接口文档已按新版原型冻结，覆盖首页、项目、任务、数据、模型、安装初始化、运行环境、个人设置和本地文件能力。前端后续接入必须以该后端文档为准，避免继续沿用 mock 字段、旧接口编号或前端自定义临时契约。

## 目标

- 以后端新版接口文档作为前端业务 API 的唯一对接来源。
- 在实现前完成 OpenSpec 提问优化，先明确接口边界、缺口、优先级和验收方式。
- 输出可执行的前端接入阶段：契约层、服务层、页面替换、风险确认、验收。
- 标记所有需要后端、产品或测试确认的问题，确认前不把临时假设写成正式实现。

## 非目标

- 本 Change 不立即实现全部接口接入。
- 不修改后端接口文档。
- 不新增未在后端文档中定义的业务接口。
- 不绕过后端执行本地路径、进程、环境、清理、下载或安装动作。

## 影响模块

- 首页：工作台、本机资源、最近操作、清理、本地工具。
- 项目：列表、规格、创建、详情、删除。
- 任务：列表、规格、预检、创建、详情、日志、输出、报告、启动、停止、删除、复制、重试、部署测试。
- 数据：列表、导入、详情、画像、检查、绑定项目、删除。
- 模型：列表、导入、详情、检查、绑定项目、删除、候选模型、下载任务。
- 设置：运行环境、安装初始化、个人信息、偏好配置。
- 系统能力：本地目录选择和路径校验。
- AI 助手：只通过后端业务接口发起真实动作，不直接替代业务 API。

## 风险

- 接口数量多，直接全量实现容易产生字段遗漏和页面状态不一致。
- 高风险接口涉及启动任务、停止任务、删除、清理、依赖安装、模型下载，必须统一确认和幂等处理。
- 当前前端仍有多个模块使用 mock 数据，替换时需要避免半真半假状态。
- 后端文档中部分接口和现有前端文档仍存在差异，需要先确认后再实现。

## 验收标准

- 待确认问题已整理成可逐项回答的问题清单。
- 每个前端模块都能映射到后端接口章节和接入阶段。
- 高风险接口的确认、幂等、错误映射和 loading 状态有统一规则。
- 未确认接口不进入实现任务。
