# 匹配附件文件和文件夹类型

## 背景

首页 AI 对话附件当前只对 Markdown 文件做了专门匹配，其他文件和文件夹都落到默认 `FILE` 展示。用户希望文件夹和对应常见文件类型都能匹配上，避免只有 `.md` 有明确标识。

## 目标

- 附件选择支持文件夹。
- 首页附件图标根据文件夹、代码、数据、文档、图片、压缩包、模型等常见类型显示对应短标签和样式。
- 未识别类型继续使用默认文件样式。

## 非目标

- 不读取文件夹内容。
- 不上传文件或文件夹内容到外部服务。
- 不改变 AI 对话接口协议。

## 验收标准

- 选择文件夹作为附件时，附件标签显示为文件夹类型。
- `.md` 之外的常见文件类型也能显示对应类型标签。
- 未识别扩展名仍显示默认文件标签。
