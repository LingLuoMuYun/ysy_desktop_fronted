# 优化首页资源概览侧栏

## 背景

首页点击顶部栏“本机资源概览”后会在右侧上下文栏展示资源概览与最近操作。当前侧栏整体可滚动，资源概览与最近操作区域高度关系不稳定，最近操作较多时会带动整个侧栏滚动。

## 目标

- 首页资源概览侧栏打开后，资源概览和最近操作两个盒子固定在侧栏内，不让侧栏整体滚动。
- 资源概览区域固定展示，最近操作区域内部出现滚动条用于查看更多操作记录。
- 资源概览与最近操作在右侧侧栏内按约 6.5:3.5 的高度比例分配。

## 非目标

- 不修改资源数据来源、接口或最近操作业务规则。
- 不调整右侧栏打开/关闭、拖拽宽度或响应式隐藏逻辑。
- 不改变其他模块的 AI 助手面板布局。

## 影响模块

- `frontend/src/layouts/LocalResourcePopover.tsx`
- `frontend/src/styles/globals.css`

## 风险等级

低风险。该变更只影响首页右侧资源侧栏的前端布局和滚动容器，不执行本机动作，不修改接口、状态机或 AI 行为。

## 验收标准

- 首页资源概览侧栏打开后，外层侧栏不出现整体滚动。
- 资源概览盒子固定在上方，不随最近操作滚动。
- 最近操作盒子固定在下方，列表区域可独立滚动查看更多记录。
- 两个盒子在侧栏内约按 6.5:3.5 分配高度，资源概览内容完整展示。
- TypeScript 检查通过。
