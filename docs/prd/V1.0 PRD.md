## 2\. 产品目标

桌面智算帮助用户在一台本机上完成 AI 项目管理、数据导入与检查、训练任务执行、推理服务创建、模型资产沉淀、运行环境管理和 AI 辅助诊断。

核心目标：

- 建立清晰的业务闭环：项目 \-\> 数据 / 环境 / 代码目录 \-\> 任务 \-\> 模型资产 \-\> 推理服务。

- 将复杂执行配置绑定在任务层，项目只保存长期上下文和资源关系。

- 让 AI 助手基于当前页面、当前对象和当前项目解释问题、生成方案、辅助填写和诊断异常。

- 所有会改变本机任务、服务、文件或配置状态的动作必须经过用户确认（或者用户给予AI所有的权限）。

## 3\. 目标用户与场景

|用户|主要诉求|典型场景|
|---|---|---|
|算法工程师|快速创建实验、管理训练和模型产物|创建目标检测训练任务、查看日志、登记模型资产|
|数据科学 / 应用开发|使用本机模型完成推理服务验证|导入 Embedding / Rerank / CV 模型并创建本机推理服务|
|本地大模型使用者|微调、部署和诊断本地大模型|创建 LoRA 微调任务、检查 Adapter 与基础模型匹配关系|
|初级 AI 用户|不熟悉训练参数和环境依赖|通过 AI 助手理解字段、生成配置方案、分析失败原因|

用户目标、关键路径与成功标准：

|用户目标|关键路径|成功标准|
|---|---|---|
|完成一次本机训练|创建项目 \-\> 绑定数据和环境 \-\> 创建训练任务 \-\> 启动任务 \-\> 查看日志和输出|任务状态到达已完成，输出目录生成训练报告和模型文件|
|启动一个本机推理服务|导入或选择模型资产 \-\> 创建推理服务 \-\> 启动服务 \-\> 打开测试页|服务健康检查通过，生成可访问 endpoint，测试请求返回结果|
|判断数据是否可训练|导入数据集 \-\> 执行基础检查 \-\> 查看异常项和配置文件方案|检查结果明确为可用、存在警告或不可用，并给出具体原因|
|判断模型是否可部署|导入模型资产 \-\> 执行模型文件检查 \-\> 查看资源风险和环境建议|检查结果明确，缺失文件、基础模型关系和资源限制可定位|
|修复失败任务|选择失败任务 \-\> AI 读取日志与配置 \-\> 复制为新任务或修复环境|新任务带入修正参数，原任务记录保持不变|

## 7\. 全局交互规则

### 7\.1 操作确认分级

|分级|定义|动作示例|交互要求|
|---|---|---|---|
|低风险|只读取信息，不改变本机状态|查看详情、搜索、筛选、展开日志、AI 解释字段|不弹确认|
|中风险|新增或修改产品内记录，但不启动进程、不改文件|创建项目、创建训练任务、创建推理服务、导入数据集记录、导入模型资产记录、导入本地环境记录、设默认运行环境、设默认 AI 模型、添加 / 编辑 AI 模型配置|提交前展示摘要确认；用户确认后写入|
|高风险|会启动 / 停止进程、占用资源、修改环境、清理文件或删除记录|启动任务、停止任务、删除任务、创建系统环境、创建自定义环境、修复环境、安装依赖、清理本机文件、删除数据集记录、删除模型资产记录、覆盖配置|必须弹高风险确认|

确认弹窗字段：

- 操作名称

- 影响对象

- 资源影响

- 是否会启动或停止本机进程

- 是否会修改文件、环境或产品记录

- 是否可撤销

- 确认与取消动作

规则：

- AI 生成的动作也必须遵守同一分级。

- 检测、重检和报告生成如果只读取文件并新增报告记录，按中风险确认；如果会写入外部目录或覆盖报告文件，按高风险确认。

- 创建环境分级：仅登记已有本地环境为中风险；创建新环境、安装依赖、修改已有环境为高风险。

- 删除操作默认只删除产品记录；如果会删除本机文件，必须单独列出文件路径并按高风险确认。

### 7\.2 本机文件与目录选择

- 首页附件上传必须调用本机文件选择器。

- 项目保存位置、数据路径、模型路径、环境路径应优先调用本机目录或文件选择能力。

- 选择后记录路径，不复制大体积数据集或模型文件。

### 7\.3 页面选择给 AI

- 用户可进入“选择页面上下文”状态。

- 可选择项目卡片、任务卡片、数据集行、模型资产行、表单字段、日志片段、异常项、资源卡片。

- 选择后 AI 助手打开，并把对象标题、类型、来源和摘要作为当前上下文。

- 用户可退出选择状态、清除上下文或更换上下文。

## 8\. 首页模块

### 8\.1 页面目标

首页是对话式工作台。用户可以通过自然语言发起项目创建、数据检查、训练任务、推理服务、环境诊断、资源分析、报告生成等，此外支持正常的AI对话功能。

### 8\.2 页面结构

- 中间：主对话框。

- 对话框内：文本输入、添加内容、本机附件选择、Plan 模式、模型选择、权限模式、当前项目选择、发送。

- 对话框下方：推荐问题。

- 右侧：本机资源概览、最近操作，可调整宽度和收起。

### 8\.3 首页附件

点击上传附件时调起本机文件管理。支持选择日志、数据配置、JSONL、YAML、模型文件、压缩包等。选择后附件名称进入当前对话上下文。支持文件拖到对话窗口读取附件。

Plan 模式：

- 默认关闭

### 8\.4 本机资源概览

展示：

- CPU

- GPU

- 内存

- 磁盘可用空间

- 网络速度

- 可清理空间

操作：

- 资源分析

- 清理前检查

- 隐藏 / 展开右侧资源区域

### 8\.5 最近操作

展示最近操作的时间和操作内容。首页和项目详情中的操作记录保持一致。

项目不承载训练脚本入口、启动命令、输出目录等任务执行字段。

- 项目状态

- 项目描述

- 工作空间

- ~~数据集数量或样本量~~

- 更新时间

- 最近任务状态

### 10\.6 任务执行器契约

训练任务启动时，后端 / 客户端执行器必须按以下契约处理：

|环节|要求|
|---|---|
|启动前校验|校验项目工作空间、代码目录、工作目录、配置文件、输出目录、数据路径和环境 ID 是否存在且可访问|
|解释器选择|使用任务绑定环境的 Python 解释器；不允许使用系统默认 Python 替代|
|命令执行|在 `workdir` 中执行 `command`，注入任务 ID、输出目录、必要环境变量|
|进程记录|启动成功后记录 `pid`、启动时间、执行命令摘要和日志引用|
|日志采集|分别采集 stdout、stderr，按时间写入日志存储；页面支持分页读取|
|停止语义|停止任务先发送正常终止信号；超时后强制结束进程；记录停止原因|
|退出码处理|exitCode = 0 且产物检查通过进入已完成；非 0 按错误类型进入失败或异常|
|输出扫描|任务结束后扫描输出目录，登记报告、checkpoint、指标文件和模型资产候选|
|失败映射|将环境不可用、数据不可用、命令不存在、依赖缺失、CUDA OOM、磁盘不足、用户停止映射为明确 errorCode|

常见 errorCode：

|errorCode|触发条件|状态|
|---|---|---|
|ENV\_NOT\_READY|Python、框架、CUDA 或依赖检查失败|启动失败|
|DATA\_NOT\_READY|数据路径、标注文件或 schema 检查失败|启动失败|
|MODEL\_NOT\_READY|推理服务模型文件或基础模型关系检查失败|启动失败|
|COMMAND\_NOT\_FOUND|脚本或命令不存在|启动失败|
|DEPENDENCY\_MISSING|运行时依赖缺失|失败|
|CUDA\_OOM|显存不足|失败|
|DISK\_NOT\_ENOUGH|输出目录或缓存目录空间不足|启动失败 / 失败|
|USER\_STOPPED|用户确认停止任务|已停止|
|PROCESS\_CRASHED|进程异常退出且无法归类|异常|

### 10\.7 推理服务运行契约

推理服务启动时，执行器必须按以下规则处理：

|环节|要求|
|---|---|
|端口检查|启动前检查端口是否占用；占用时进入启动失败并提示冲突进程信息|
|启动方式|根据服务类型生成真实启动命令，例如 HTTP API、OpenAI 兼容接口或自定义命令|
|endpoint 格式|本机访问使用 `http://127.0.0.1:{port}/{path}`；局域网访问使用本机局域网 IP|
|健康检查|启动后轮询健康检查地址；连续失败达到阈值进入启动失败|
|进程管理|记录服务 pid、端口、模型资产 ID、启动命令摘要、日志引用|
|统计来源|请求次数、平均延迟、错误率来自服务网关或测试页请求记录|
|停止服务|停止时释放端口和进程；停止失败进入异常并提示手动处理建议|
|进程崩溃|运行中进程退出时进入异常，保留最近日志和退出码|

本机版本的副本数规则：

- 默认副本数为 1。

- 本版本只支持单副本本机服务；用户输入大于 1 时，表单必须提示不支持并阻止提交。

- 不自动分配多个端口，不启动多个服务进程。

健康检查默认参数：

|字段|默认值|说明|
|---|---|---|
|path|`/health`|用户可在表单中修改|
|interval|5s|轮询间隔|
|timeout|3s|单次请求超时|
|retries|12|连续失败达到次数后进入启动失败|

### 15\.2 数据检查规则

|数据类型 / 格式|必要检查|警告判定|不可用判定|
|---|---|---|---|
|目标检测 / YOLO|`images/train`、`images/val`、`labels/train`、`labels/val`、`data.yaml` 可读；图片与标签匹配|少量标签缺失、类别分布偏斜、空标注比例偏高|关键目录缺失、data\.yaml 不可读、标签格式无法解析|

|Embedding / JSONL|包含 query、positive，可选 negative|负样本不足、重复 query 较多|query 或 positive 缺失|
|Rerank / JSONL|包含 query、document、label|同组样本不足、正负样本比例异常|label 缺失或无法解析|
|自定义|manifest 或 schema 可读|schema 不完整、字段角色不明确|manifest / schema 缺失且无法推断|

- 创建训练任务前发现数据检查时间早于数据记录更新时间。

### 15\.3 模型检查规则

|模型类型|必要检查|警告判定|不可用判定|
|---|---|---|---|
|CV 权重|文件存在、扩展名匹配、大小可读、框架可识别|缺少类别配置、精度或输入尺寸未知|权重文件不存在、格式不支持|
|LLM 基座模型|config、tokenizer、权重分片存在；参数规模和量化信息可识别|显存可能不足、tokenizer 版本未知|config 或权重缺失，无法识别模型结构|
|LoRA / Adapter|adapter\_config、adapter 权重存在，baseModel 信息可读|基础模型未绑定、量化关系需确认|Adapter 文件缺失或基础模型不兼容|
|合并后 LLM|config、tokenizer、权重完整，推理框架可识别|上下文长度或量化信息不完整|权重缺失、格式不支持|
|Embedding|tokenizer、config、权重存在，向量维度可识别|向量维度未知、CPU 性能风险|关键文件缺失|
|Rerank|tokenizer、config、权重存在，输入 schema 可识别|Top K 或最大长度需确认|关键文件缺失|

重检触发：

- 用户点击模型检查。

- 模型路径、基础模型、推荐环境发生变化。

- 创建推理服务前模型检查结果为空或已过期。

### 15\.4 环境检测规则

|检查项|可用|存在警告 / 需修复|不可用|
|---|---|---|---|
|Python|解释器可执行，版本满足模板要求|版本可运行但不在推荐范围|解释器不存在或无法启动|
|包管理器|conda / pip / uv 等可执行|源不可达或版本偏旧|包管理器不可用|
|框架|PyTorch / Transformers / vLLM 等可导入|版本与模板推荐不一致|关键框架无法导入|
|CUDA / 驱动|CUDA 与框架匹配，GPU 可用|CUDA 可用但版本不推荐，或仅 CPU 可用|训练 / 推理所需 GPU 不可用|
|磁盘 / 内存|满足任务最低要求|接近最低阈值|低于最低阈值|

导入本地环境时，框架和 CUDA 信息必须由检测得出，不能要求用户手填。

### 15\.5 AI 动作边界

AI 输出分为三类：

|类型|能力|执行边界|
|---|---|---|
|explain|解释页面、字段、日志、异常和资源|只读，不修改任何状态|
|field\_suggestion|生成表单填写建议、参数建议、报告结构建议|用户点击应用或提交后才写入|
|action\_request|生成受控动作请求，如启动任务、停止任务、检测环境、生成报告|必须进入对应真实操作入口，并按风险分级确认后执行|

权限模式：

|权限模式|默认|能力|
|---|---|---|
|只读建议|否|只允许 explain|
|辅助填写|是|允许 explain、field\_suggestion|
|确认后执行|否|允许 explain、field\_suggestion、action\_request|

AI 动作请求结构：

|字段|说明|
|---|---|
|actionType|explain、field\_suggestion、action\_request|
|targetModule|首页、项目、任务、数据、模型、设置|
|targetId|目标对象 ID|
|intent|用户意图|
|proposedChanges|建议修改的字段和值|
|requiredConfirmation|需要用户确认的风险点|
|riskLevel|low、medium、high|

规则：

- AI 不直接绕过页面按钮或后端权限执行动作。

- 用户确认后，由产品后端执行真实动作，并将成功或失败结果回写到 AI 对话和业务页面。

- AI 失败时必须展示失败原因、已读取上下文和可重试建议。

### 15\.6 报告生成规格

|报告类型|触发方式|内容结构|产物|
|---|---|---|---|
|训练完成报告|训练任务已完成后提示用户生成，用户确认后生成|任务信息、数据集、环境、训练参数、指标摘要、输出模型、日志摘要、后续建议|Markdown / HTML，记录 reportId 和路径|
|失败诊断报告|任务失败、启动失败或异常后用户触发|错误摘要、日志证据、错误分类、影响范围、建议修复步骤、复制为新任务建议|Markdown / HTML，关联原任务|
|推理服务报告|推理服务运行后用户触发|服务配置、模型资产、endpoint、健康状态、请求统计、错误率、资源占用|Markdown / HTML，关联推理服务|

报告规则：

- 报告默认写入任务输出目录下的 `reports/`。

- 如果输出目录不可写，报告状态为生成失败，并提示用户选择可写目录。

- 生成报告会新增 report 记录，不覆盖已有报告。

- 报告生成失败不改变任务主状态。

- 报告有独立状态：待生成、生成中、已生成、生成失败；不占用任务 phase。

## 16\. 核心业务流程

### 16\.1 创建项目并创建训练任务

1. 用户在首页输入目标，或在项目模块点击创建项目。

2. 系统创建项目表单，用户填写基础信息、保存位置、数据集、运行环境。

3. 用户确认创建项目。

4. 进入项目详情，点击创建训练任务。

5. 训练任务表单根据项目类型带入训练类型、数据集、环境和执行入口建议。

6. 用户修改参数并创建任务。

7. 任务状态为待启动。

8. 用户确认启动任务。

9. 系统执行检查并启动训练，采集日志、指标和输出。

10. 训练完成后用户登记模型资产。

### 16\.2 从模型资产创建推理服务

1. 用户在模型模块导入或选择模型资产。

2. 系统检查模型文件、基础模型关系、资源需求和推荐环境。

3. 用户点击创建推理服务。

4. 进入推理服务表单，自动带入模型资产、服务类型和推荐环境。

5. 用户确认端口、访问范围、并发、超时和服务参数。

6. 创建任务后状态为待启动。

7. 用户确认启动任务。

8. 系统检查端口、模型文件、推理依赖和资源。

9. 启动成功后生成服务地址。

10. 用户进入测试页验证响应。

### 16\.3 复制任务为新任务

1. 用户在任务详情点击复制为新任务。

2. 系统打开任务表单并带入原任务配置。

3. 用户修改任务名称、数据集、环境、执行入口或参数。

4. 用户点击创建新任务。

5. 系统创建新任务，状态为待启动。

6. 原任务保持不变。

### 16\.4 AI 诊断失败任务

1. 任务失败或异常。

2. 用户选择日志片段或点击 AI 诊断。

3. AI 读取任务配置、日志、环境摘要、数据或模型上下文。

4. AI 输出问题摘要、原因、日志证据、影响范围、修复步骤。

5. 用户选择调整参数并复制为新任务，或修复环境。

6. 高风险动作执行前确认。

### 16\.5 导入并检查数据集

1. 用户进入数据模块，点击导入数据集。

2. 用户选择本机数据路径，填写数据类型、格式、标注文件、关联项目。

3. 用户确认导入数据集记录。

4. 系统记录路径和元信息，不复制数据文件。

5. 如果勾选导入后检查，系统执行数据检查。

6. 检查结果写入数据集状态、检查摘要和异常项。

7. 用户可在数据详情中重新检查、绑定项目或让 AI 分析异常。

失败处理：

- 路径不存在：不允许导入或导入后状态为不可用，需展示具体路径。

- 检查失败：记录 errorCode / errorMessage，不影响已创建的数据集记录。

### 16\.6 导入并检查模型资产

1. 用户进入模型模块，点击导入模型。

2. 用户选择本机模型文件或目录，填写模型类型、来源、关联项目。

3. 若模型为 LoRA / Adapter 或合并模型，用户填写或选择基础模型。

4. 用户确认导入模型资产记录。

5. 系统只记录路径和元信息，不复制模型文件。

6. 如果勾选导入后检查，系统执行模型检查。

7. 检查结果写入模型状态、检查结论、推荐环境和资源风险。

8. 可用或存在警告的模型可以进入创建推理服务流程。

失败处理：

- 模型路径不存在：状态为不可用。

- 基础模型不匹配：状态为存在警告或不可用，依据是否阻断部署。

- 资源不足：状态为存在警告，创建推理服务前再次提示。

### 16\.7 创建或导入运行环境

1. 用户进入设置 / 环境创建。

2. 用户选择创建系统环境、导入本地环境或自定义环境。

3. 系统按对应方式展示字段。

4. 用户确认创建或导入。

5. 系统登记环境记录，并执行检测。

6. 检测结果写入环境状态和检查项。

7. 需修复时，用户可点击修复环境并进行高风险确认。

失败处理：

- 导入路径不可访问：不创建环境记录或创建后标记不可用，必须给出路径原因。

- 依赖安装失败：环境状态为需修复，保留安装日志。

- 修复失败：状态回到需修复或不可用，展示失败项和建议。

### 16\.8 添加并检测 AI 助手模型

1. 用户进入设置 / AI 助手配置。

2. 用户点击添加模型，填写模型名称、厂商、API Base URL、API Key、模型标识、上下文长度、最大输出长度、Temperature。

3. 用户保存配置。

4. 系统脱敏保存 API Key。

5. 用户执行连接测试。

6. 连接成功后状态为可用，可设为默认模型。

7. 点击 AI 助手模型卡片可查看详情。

失败处理：

- API Key 缺失或无效：状态为异常，详情展示连接错误。

- API Base URL 不可达：状态为异常，提示网络或地址问题。

- 模型标识不存在：状态为异常，提示检查模型标识。

## 17\. 状态流转

### 17\.1 任务状态

任务只允许一个主状态。环境检查、数据检查、模型检查、资源检查、验证评估、导出产物属于 `phase`，不作为主状态枚举。

主状态定义：

|主状态|含义|可执行动作|
|---|---|---|
|等待配置|必填配置缺失|编辑|
|待启动|配置完整，等待用户启动|启动、编辑、复制、删除|
|检查中|已确认启动，正在执行启动前检查|查看日志|
|排队中|检查通过，等待本机资源或执行器调度|查看日志|
|运行中|训练进程或推理服务正在运行|停止、复制|
|停止中|用户已确认停止，执行器正在释放资源|查看日志|
|已停止|用户停止成功|启动、复制、删除|
|已完成|训练任务正常完成并完成产物扫描|复制、生成报告、删除|
|启动失败|启动前检查或服务启动失败|启动、编辑、复制、删除|
|失败|运行中发生可归因错误|复制、生成诊断报告、删除|
|异常|进程崩溃或无法归因错误|复制、生成诊断报告、删除|

推理服务状态规则：服务启动并通过健康检查后保持运行中；用户停止后进入已停止；服务进程崩溃进入异常；健康检查通过不进入已完成。

阶段进度定义：

|phase|适用对象|说明|
|---|---|---|
|环境检查|训练 / 推理服务|检查运行环境|
|数据检查|训练|检查数据集|
|模型检查|推理服务|检查模型资产|
|资源检查|训练 / 推理服务|检查 GPU、内存、磁盘、端口|
|服务启动|推理服务|启动服务进程并等待健康检查|
|训练中|训练|训练进程运行|
|验证评估|训练|执行验证或评估|
|导出产物|训练|导出 checkpoint、模型或指标|

### 17\.2 数据、模型、环境状态

|对象|状态|
|---|---|
|数据集|未检查、检查中、可用、存在警告、不可用|
|模型资产|未检查、检查中、可用、存在警告、不可用、已关联推理服务|
|运行环境|未检测、检测中、可用、需修复、修复中、不可用、创建中|
|AI 模型配置|待检测、可用、异常|

## 19\. 验收标准

### 19\.1 首页

- 能通过居中对话框输入自然语言。

- 上传附件调起本机文件管理。

- 对话框下方可选择模型、权限模式和当前项目。

- 推荐问题可点击并进入对话或生成计划。

- 右侧资源卡片可隐藏、恢复和调整宽度。

- 首页不出现 Git 仓库、分支、commit、diff 等管理能力。

### 19\.2 项目

- 创建项目表单不要求脚本入口。

- 项目详情操作记录只展示时间和操作内容，最新在最上面。

- 项目详情模型资产查看在当前页弹出详情。

- 项目模块入口使用创建推理服务文案。

### 19\.3 任务

- 待启动任务有启动按钮。

- 训练任务和推理服务详情顶部操作一致。

- 删除任务使用图标并二次确认。

- 复制为新任务打开可编辑表单，提交后直接创建新任务。

- 任务概览中任务状态摘要位于任务信息下方。

- 任务详情不展示独立运行前检查卡片。

### 19\.4 数据

- 支持导入本地数据集并记录路径。

- 支持绑定项目和重新检查。

- 高级信息 / 检查依据默认折叠。

- AI 可基于异常项生成检查摘要和配置文件方案。

### 19\.5 模型

- 导入模型只记录本机路径，不复制大模型文件。

- 外部来源场景提供适合本机资源的小模型选择。

- 模型资产行和详情只展示资产字段，不展示 AI 连接字段或能力标签。

- 模型详情展示检查结论和推理服务建议。

- 从模型创建服务后，运行状态进入任务模块。

### 19\.6 设置

- 环境创建有创建系统环境、导入本地环境、自定义环境三种方式。

- 导入本地环境不要求填写框架和 CUDA 版本。

- AI 助手配置只展示模型管理、连接检测和默认模型设置。

- 添加 AI 模型表单不展示流式输出、工具调用、安全审计、Top P。

- 点击 AI 助手模型卡片打开详情。

### 19\.7 全局 AI 助手

- 右侧 AI 助手支持新建对话和最近对话列表。

- 当前上下文以轻量条展示，可展开详情。

- 推荐问题出现在对话消息中。

- 输入区下方支持模型选择、权限选择和项目范围选择。

- 附加页面元素后，AI 能基于该对象回答。

### 19\.8 真实能力验收用例

|用例|输入 / 前置条件|期望结果|
|---|---|---|
|导入有效 YOLO 数据集|选择包含 images、labels、data\.yaml 的目录|数据集创建成功，检查结果为可用或存在警告，展示具体检查项|
|导入无效数据路径|选择不存在的路径|不允许进入可用状态，展示路径不存在错误|
|创建训练任务并启动|选择可用项目、数据集、环境和可执行命令|任务进入检查中、排队中、运行中；记录 pid、日志和启动时间|
|训练命令不存在|command 指向不存在脚本|任务进入启动失败，errorCode 为 COMMAND\_NOT\_FOUND|
|训练 CUDA OOM|训练进程日志出现 CUDA out of memory 并非 0 退出|任务进入失败，AI 诊断能引用日志证据|
|停止运行中任务|运行中任务点击停止并确认|任务进入停止中，进程结束后进入已停止，记录 USER\_STOPPED|
|推理服务端口冲突|端口已被其他进程占用|服务启动失败，提示冲突端口和进程信息|
|推理服务健康检查失败|服务进程启动但健康检查连续失败|服务进入启动失败，保留启动日志|
|模型 Adapter 缺基础模型|导入 LoRA / Adapter 未配置基础模型|检查结果为存在警告或不可用，并阻止直接启动不兼容服务|
|导入本地环境|选择已有 Conda 或 Python 解释器|系统检测 Python、框架、CUDA，不要求用户手填框架和 CUDA|
|环境缺少依赖|任务启动前环境检测无法导入关键包|任务进入启动失败，提示 DEPENDENCY\_MISSING|
